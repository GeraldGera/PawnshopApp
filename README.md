# PawnshopApp

Учебная программа для автоматизации работы ломбарда. Написана на C++ с использованием графического интерфейса Qt Widgets и базы данных PostgreSQL. Проект сделан в рамках практической части курсовой работы.

В приложении разделены интерфейс и логика (архитектура MVC), есть рабочее окно входа и регистрации, проверка ролей (Администратор / Пользователь) и полноценное управление данными через таблицы.

## Технологии

* **Язык:** C++17
* **Интерфейс:** Qt Widgets
* **База данных:** PostgreSQL (подключение через модуль Qt SQL и драйвер QPSQL)
* **Сборщик проекта:** CMake
* **Контроль версий:** Git

## Как устроен код (Архитектура)

Программа разделена на три логические части, чтобы код не превращался в кашу:

* `src/Model` (Модель) — отвечает за работу с данными. Здесь настраивается подключение к базе данных, лежат классы-сущности для таблиц, выполняются все SQL-запросы и хэшируются пароли пользователей. Интерфейс сюда напрямую не лазит.
* `src/Controller` (Контроллер) — это «мозг» программы. Он принимает команды от пользователя, проверяет права доступа, обрабатывает логику авторизации и передает очищенные данные между базой и экраном.
* `src/View` (Представление) — визуальная часть. Здесь лежат формы, окна входа и регистрации, личный кабинет, а также таблицы для просмотра и редактирования залогов.

## База данных

 Клиенты  `clients`  Личные данные посетителей 
 Профили  `profiles`  Учетные записи для входа сотрудников 
 Договоры  `contracts` Документы об оформлении займа 
 Залог `pledges`  Информация о заложенном имуществе 
 Товары  `products`  Описание сданных вещей 
 Цены  `prices` История изменения стоимости и оценки 
 Продажи  `sales`  Данные о выкупленных или невыкупленных вещах 

При запуске скрипта `sql/init.sql` автоматически создаются все связи между таблицами (внешние ключи), проверки на пустые поля (`NOT NULL`), уникальность и создаются базовые индексы для быстрой работы.

## Права доступа (Роли)

В системе заложены две роли с разными возможностями:

1. **Администратор (`admin`)** — имеет полный доступ к программе. Может добавлять, изменять, просматривать и удалять записи во всех таблицах (клиенты, залоги, договоры, продажи).
2. **Обычный пользователь / Сотрудник (`user`)** — имеет доступ только к своему личному кабинету, может редактировать свои данные и просматривать договоры, залоги и продажи, к которым он привязан.

### Данные для проверки работы (Тестовые аккаунты):

| Логин | Пароль | Роль в системе |
| --- | --- | --- |
| `admin` | `Admin123!` | Администратор |
| `user` | `User123!` | Пользователь |

### Безопасность:

* Пароли в базе данных не хранятся в открытом виде — они зашифрованы с помощью надежного алгоритма `pbkdf2_sha256` с добавлением случайной «соли».
* При регистрации работает строгая проверка пароля: он должен быть не короче 8 символов, содержать хотя бы одну цифру, одну заглавную букву и один специальный знак (например, `!`, `@`, `#`).
