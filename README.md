# PawnshopApp

Учебное desktop-приложение для ломбарда на C++/Qt Widgets с базой данных PostgreSQL. Проект реализует практическую часть курсовой работы: MVC-структуру, ORM-подход через классы сущностей, регистрацию и авторизацию, ролевую модель доступа и CRUD через GUI.

## Стек

- C++17
- Qt Widgets
- Qt SQL, драйвер QPSQL
- PostgreSQL
- CMake
- Git

## Архитектура MVC

- `src/Model` - сущности, репозитории, Singleton подключения к БД, хэширование паролей.
- `src/Controller` - сценарии авторизации, профиля и CRUD-операций с проверкой прав.
- `src/View` - окна Qt Widgets: вход, регистрация, личный кабинет, таблицы и формы.

SQL-запросы находятся только в Model-слое. View не работает с `QSqlQuery` напрямую: данные проходят через контроллеры и объекты сущностей.

## База данных

Имена таблиц и полей в PostgreSQL сделаны в `ASCII snake_case`, чтобы не зависеть от кавычек и кодировок. Соответствие схеме из курсовой:

| Схема | PostgreSQL |
| --- | --- |
| Клиенты | `clients` |
| Профили | `profiles` |
| Договоры | `contracts` |
| Залог | `pledges` |
| Товары | `products` |
| Цены | `prices` |
| Продажи | `sales` |

Скрипт `sql/init.sql` создает первичные и внешние ключи, `NOT NULL`, `UNIQUE`, `CHECK`, индексы и тестовые записи.

## Роли

- `admin` - полный CRUD по клиентам, профилям, договорам, залогу, товарам, ценам и продажам.
- `user` - личный кабинет: изменение своих регистрационных данных и просмотр своих договоров, залога, товаров, цен и продаж.

Тестовые пользователи после выполнения `sql/init.sql`:

| Логин | Пароль | Роль |
| --- | --- | --- |
| `admin` | `Admin123!` | `admin` |
| `user` | `User123!` | `user` |

Пароли хранятся в формате `pbkdf2_sha256:iterations:salt:hash`, не открытым текстом. Сложность пароля проверяется регулярным выражением: минимум 8 символов, цифра, заглавная буква и спецсимвол.
